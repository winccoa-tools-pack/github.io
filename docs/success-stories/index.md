# Success Stories

Short write-ups showing how teams use these tools to improve quality, automation, and delivery speed.

## 1) VS Code adoption in a niche industry

Our top VS Code extension has passed **1600+ downloads** since the official first release on **31 January**.
That number might look small in general software, but for **industrial SCADA** it’s a strong validation that modern WinCC OA tooling in VS Code is valuable.

- Extension: https://marketplace.visualstudio.com/items?itemName=RichardJanisch.winccoa-project-admin

## 2) A lightweight quality gate, adopted fast

Our youngest npm package, **@winccoa-tools-pack/npm-winccoa-syntax-check**, reached **94 downloads in 8 days**.
It’s a simple package that provides **basic static analysis** for WinCC OA projects — and it shows how important this quality gate is for WinCC OA customers and integrators.
You can add it easily to your CI/CD pipeline or run it locally.

- Tool docs: [npm-winccoa-syntax-check](/docs/tools/npm/npm-winccoa-syntax-check)
- npm: https://www.npmjs.com/package/@winccoa-tools-pack/npm-winccoa-syntax-check

## 3) AI integration via MCP

Our MCP server closes the gap by enabling **WinCC OA engineering inside the VS Code editor**, so AI assistants and editor automation can integrate with real project context.

- Tool docs: [winccoa-mcp-server](/docs/tools/other/winccoa-mcp-server)
